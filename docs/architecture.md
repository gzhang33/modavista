# 项目架构设计文档

## 整体架构

Dreamoda 电商平台采用前后端分离架构，实现关注点分离和模块化设计。

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   前端应用      │    │   后端服务      │    │   共享资源      │
│  (React + Vite) │◄──►│   (PHP + API)   │◄──►│   (Schemas)     │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

## 目录结构说明

### 前端架构 (frontend/)
- **技术栈**: React 18 + TypeScript + Vite + TailwindCSS
- **组件化**: 基于 Radix UI 的组件库
- **状态管理**: React Query + Context API
- **路由**: Wouter (轻量级路由)

### 后端架构 (backend/)
- **技术栈**: PHP + MySQL
- **API设计**: RESTful API
- **模块划分**:
  - `admin/`: 管理后台界面和逻辑
  - `api/`: 对外API接口
  - `config/`: 配置文件

### 共享资源 (shared/)
- **数据结构**: TypeScript 类型定义
- **接口规范**: API 请求/响应格式
- **数据库模型**: 数据表结构定义

## 模块交互流程

1. **用户访问**: 前端应用加载，通过路由分发到对应页面
2. **数据请求**: 前端通过 API 客户端向后端请求数据
3. **业务处理**: 后端接收请求，处理业务逻辑，访问数据库
4. **数据返回**: 后端返回结构化数据给前端
5. **界面渲染**: 前端接收数据，更新组件状态，渲染界面

## 技术选型理由

- **前端**: React + Vite 提供快速的开发体验和构建性能
- **后端**: PHP 成熟稳定，适合快速开发和部署
- **数据库**: MySQL 关系型数据库，支持复杂查询
- **样式**: TailwindCSS 提供快速样式开发

## 扩展性设计

- 模块化架构支持独立开发和部署
- 共享资源确保前后端数据结构一致性
- 配置文件分离便于环境管理
- API 设计支持版本控制和向后兼容
